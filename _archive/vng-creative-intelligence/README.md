# VNG P5 Creative Performance Intelligence

Deployable AABW 2026 scaffold for the VNG creative-intelligence brief.

The app keeps the two halves separate:

- Analysis: `join_perf_ltv` -> `tag_themes` -> `cluster` on high-value-player share -> `recommend_direction`.
- Generation: `gen_skins(theme)` -> `simulate_reception` through `getSimClient()` -> `pick_best` -> `to_3d`.

Everything runs local-and-mock by default and progressively upgrades to real services as keys appear:

- `data/ads.sample.csv` is a deterministic synthetic ad export generated by `npm run data:gen`: 42 creatives / 10 themes across history + holdout splits, with plausible funnel invariants and a documented `hypercasual-candy` D7-ROAS trap theme.
- `tag_themes` reads theme labels from export metadata by default, and runs a real `CHEAP_MODEL` vision pass over generated PNG thumbnails when `AI_GATEWAY_API_KEY` or `VERCEL_OIDC_TOKEN` is set (`lib/vision.ts`). Every tag reports its `source` and any fallback reason.
- `simulate_reception` uses `MockSimClient` until `MIROSHARK_URL` is set, at which point the real `MiroSharkClient` (HTTP, auth, timeout, `verdict.json` mapping) takes over (`lib/sim-client.ts`).
- `gen_skins` -> Nano Banana and `to_3d` -> Meshy are adapters (`lib/gen-adapters.ts`) that render/build live when `NANO_BANANA_KEY` / `MESHY_KEY` are present, and otherwise return checked-in PNG concepts + `public/models/sample-skin.glb`. Each result reports its `source` and fallback reason.
- Supabase persistence is optional. With the starter Supabase env vars present, page visits persist snapshots and the dashboard can read past runs back.

None of these are required to run: with no keys the app deploys green on labeled fallbacks.

## Run

```bash
npm install
npm run data:gen
npm run build
npm run eval
```

`npm run eval` prints the holdout pLTV-weighted ROAS lift from following the agent's actual top-1 recommendation, plus a baseline table for D7 ROAS, installs, and spend pickers. It also reports a stratified creative-level bootstrap 95% CI and a permutation-test p-value.

## Demo Surface

The first screen is the working dashboard:

- theme clusters and a pLTV-weighted ROAS chart via Recharts, labeled with the live tagging source
- CSV upload for bring-your-own ad exports using the same parser and validator as the sample data
- optional past-run history from Supabase
- recommendation panel with evidence rank and an avoid-overfunding note, without fabricated confidence badges
- holdout backtest comparison against portfolio, rejected themes, and naive baselines, with CI and p-value on-panel
- skin generation button with real/fallback service badges for Nano Banana, MiroShark, and Meshy
- surfaced AI tool steps in the analysis-agent panel, with a clear "set `AI_GATEWAY_API_KEY`" state when no key is present
- `<model-viewer>` preview of the generated/bundled GLB
